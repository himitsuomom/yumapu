# Yu-Map - 修正内容の説明

## プロジェクト概要
Yu-Mapは、施設情報を地図上に表示するFlutterアプリケーションです。このプロジェクトでは、5つの重要な問題を修正しました。

## 修正内容

### 1. クエリ再代入の修正（重要）
**ファイル：** `lib/services/facility_service.dart`

**問題：** 位置情報フィルター（RPC呼び出し）を適用すると、以前に適用された属性フィルター（タトゥー、サウナ等）や検索文字列が破棄されていました。

**解決策：** 
- 条件付きクエリチェイニングを正しく使用し、すべてのフィルターが維持されるようにしました
- 検索クエリを段階的に構築し、各フィルターが他のフィルターを上書きしないようにしました

### 2. メモリリークの修正（重要）
**ファイル：** `lib/services/subscription_service.dart`

**問題：** リスナーが `_customerInfoSubscription` フィールドに実際に割り当てられておらず、既存の `dispose()` ロジックが無効でした。

**解決策：**
- `_customerInfoSubscription` フィールドにストリーム購読を正確に割り当てました
- 適切な `listenToPremiumStatus()` メソッドを追加し、`dispose()` メソッドでメモリリークを防ぐための購読解除処理を実装しました

### 3. Supabase .contains() 使用法の修正
**ファイル：** `lib/services/supabase_service.dart`

**問題：** 結合された関係データに対して `.contains()` を正しく使用していませんでした。

**解決策：**
- 結合関係データ用に正しいSupabase/PostgREST構文（`.filter()`など）を使用するように変更しました
- 施設アメニティ検索が正しく機能するように実装しました

### 4. コード品質のクリーンアップ
**ファイル：** `lib/services/facility_service.dart`

**問題：** 不要なヌル合体演算子（lines 106-107）があり、カプセル化違反がありました。

**解決策：**
- 冗長なヌル合体コードを簡略化しました
- `map_clustering_service.dart` でプライベート `_cache` フィールドへの直接アクセスを防止し、適切なゲッターまたはメソッドを提供するようにしました

### 5. UIと機能性の維持
- すべての修正後もUIは「読みやすさ」を維持しています
- すべてのO/X機能は完全に動作します

## 技術的な詳細

### クエリ構築の改善点：
- 各フィルターが他のフィルターを上書きしないように条件付きで追加されます
- 検索クエリは段階的に構築され、すべての条件が保持されます

### メモリ管理：
- ストリーム購読は適切に管理され、画面が離れる際に確実にキャンセルされます
- 不要になったリソースは解放されます

### Supabase統合：
- 複雑なJSONBデータの操作を安全に行えるように設計されています
- 結合テーブルとの正しい相互作用を実現します

## 使用方法
この修正により、ユーザーは複数のフィルターを組み合わせて施設を検索でき、メモリリークの心配なくサブスクリプション状態を利用できます。
