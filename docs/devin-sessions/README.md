# Devin Session Retrospectives

This directory stores retrospective notes from Devin sessions working on Yu-Map.

## Purpose

Retrospectives capture lessons learned from complex or retry-heavy tasks. They feed into the continuous improvement process defined in `.devin/knowledge/07-continuous-improvement.md`.

## When to Write a Retrospective

- After a complex task involving multiple retries or unexpected blockers.
- After discovering a missing pattern or undocumented convention.
- After a task that revealed a gap in knowledge or playbook files.

## Template

Use `RETROSPECTIVE_TEMPLATE.md` in this directory.

## File Naming

Name retrospective files as: `YYYY-MM-DD_short-description.md`

Example: `2024-03-15_fix-query-filter-chain.md`

## What Belongs Here vs Knowledge/Playbooks

| Content | Location |
|---|---|
| Repo-wide reusable facts | `.devin/knowledge/` |
| Repeatable task procedures | `.devin/playbooks/` |
| Session-specific lessons and context | `docs/devin-sessions/` |
